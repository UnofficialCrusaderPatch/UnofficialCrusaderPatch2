Liste aller Änderungen

We hope you have all had a happy Easter.
As a special festive treat, there is finally a new UCP release created by a whole team of new developers and Crusader fans!

List of all changes

Änderungen

Folgende Änderungen und Verbesserungen wurden mit Version 2.14 des Inoffiziellen Crusader Patches hinzugefügt.

Wir haben jetzt eine Website und einen Discord-Server!

Hinweise für Mehrspieler:

  • Die Optionen zum Ändern der Farbe von Spieler 1 und Aktivieren des Zuschauermodus können nicht im Mehrspieler verwendet werden.
  • Alle anderen Einstellungen können im Mehrspieler verwendet werden, sofern alle Mitspieler die selbe Konfiguration verwenden.

Neue Funktionen und Verbesserungen:

Allgemein

  • Mehrspieler-Standardgeschwindigkeit

    • Mit dieser Option kannst du die Mehrspielergeschwindigkeit auf einen anderen Standardwert als 40 festlegen. Auch nach Beenden einer Partie bleibt die Standardgeschwindigkeit auf diesem Wert.

  • Vereinheitlichte Waffenreihenfolge

    • Hiermit wird die Reihenfolge vereinheitlicht, in der Waffen im Marktplatz sowie in der Waffenkammer angezeigt werden. Außerdem befinden sich Lederrüstungen nun zwischen Streitkolben und Armbrüsten, sowie Eisenrüstungen zwischen Hellebarden und Schwertern, um die Navigation im Marktplatz zu erleichtern.

  • Fehlerbehebungen für Leiterträger

    • Bei der Option für resistentere Leiterträger wurden der angezeigte Preis (vorher 4 Gold) und der tatsächliche Preis (vorher 38 Gold) beide auf den beabsichtigten Wert von 20 Gold gesetzt.
    • Die KI nutzt nun Leiterträger in Belagerungen verlässlicher - unabhängig davon, ob ihre eigene Burg geschlossen oder offen ist.

  • Feuerballisten-Bugfix

    • Wir hatten Feuerballisten zuvor bereits beigebracht, selbstständig auf Mönche zu schießen. Diese Option wurde nun erweitert, damit sie auch automatisch auf Tunnelgräber schießen.

  • Spielerfarben-Änderungen

    • In einer früheren Version wurde bereits die Möglichkeit hinzugefügt, die Farbe von Spieler 1 zu ändern. Allerdings gab es bisher noch einige UI-Elemente, bei denen die Farbe dabei nicht geändert wurde - damit ist jetzt Schluss.

  • Starttruppen-Auswahl im Identitätsmenü

    • Mit Version 2.14 des UCP ist es nun möglich, im “Identität zulegen”-Menü des Spiels die Nah- und Fernkampf-Starteinheiten für den Spieler zu ändern.
    • Hinweis: Aktuell beeinflusst diese Einstellung nur den Spielmodus Normal, aber nicht Kreuzritter oder Deathmatch, und ist nur aktiv bis das Spiel beendet wird.

  • Strongholdify

    • Für alle Oldschool-Stronghold-Liebhaber und diejenigen, die die Produktionsketten aus Stronghold 1 gewohnt sind, skaliert diese Option die hochgeschraubte Produktion in Stronghold Crusader auf Stronghold 1 Verhältnisse herunter.
    • Das wirkt sich auf die Beliebtheitsboni für Kirchen und Kathedralen sowie Bierausschank aus, ebenso wie auf Bauern-Spawnraten und die Produktionsmengen für Ressourcen (z.B. liefert ein Holzfäller damit 12 statt 18 Holz ab).

Anpassbare Starttruppen und Burgherrn-Stärke

Hinweis für Mehrspieler: Die hier getroffenen Einstellungen sind kompatibel für den Mehrspielermodus, solange sie bei allen Spielern identisch sind. Ansonsten kann es zu Abstürzen oder zur Desynchronisation kommen.

  • Starttruppen

    • Im neuen Reiter Starttruppen kann eine Starttruppen-Konfiguration ausgewählt werden, die dann für alle KI-Charaktere sowie europäische und arabische Spieler-Burgherrn verwendet wird.
    • Die Vanilla-Konfiguration sowie eine UCP-Patch-Konfiguration für Starttruppen sind im Patcher enthalten und können zur einfachen Bearbeitung exportiert werden.
    • Um eigene Konfigurationen zu erstellen, können entweder die mitgelieferten exportiert und bearbeitet werden, oder eigene in ein Unterverzeichnis des Patchers hinzugefügt werden.
    • Die exportierten Starttruppen-Konfigurationen befinden sich im Ordner resources\troops\exports. Deine eigene Starttruppen-Konfiguration (im JSON-Format und mit der Dateiendung .json) kann in den Ordner resources\troops gelegt werden und erscheint nach einem Reload im Starttruppen-Tab im Patcher.

  • Burgherrenstärke

    • Innerhalb der Starttruppen-Datei können auch Stärke, Typ und angezeigte Stärkepunkte der KI-Charaktere geändert werden. Die Stärke wird als Multiplikator für den Basis-Trefferpunktwert von 150000 (entspricht dem Burgherrn menschlicher Spieler) angegeben. Es können entweder Blaue, Gelbe oder Keine Stärkepunkte angezeigt werden, mit einer Anzahl von 0 bis 5.

Anpassbare Startgüter

Hinweis für Mehrspieler: Die hier getroffenen Einstellungen sind kompatibel für den Mehrspielermodus, solange sie bei allen Spielern identisch sind. Ansonsten kann es zu Abstürzen oder zur Desynchronisation kommen.

  • Im neuen Reiter Startgüter kann eine Startgüter-Konfiguration ausgewählt werden, die dann für alle menschlichen Spieler und KIs verwendet wird.
  • Die Vanilla-Konfiguration für Startgüter ist im Patcher enthalten und kann zur einfachen Bearbeitung exportiert werden.
  • Die exportierten Startgüter-Konfigurationen befinden sich im Ordner resources\goods\exports. Deine eigene Startgüter-Konfiguration (im JSON-Format und mit der Dateiendung .json) kann in den Ordner resources\goods gelegt werden und erscheint nach einem Reload im Startgüter-Tab im Patcher.

Installation von AIVs

  • Es wird nun jedes Mal eine Sicherungskopie deiner aktuell installierten Burgen erstellt, wenn durch den Patcher ein Burgenpaket eingefügt wird.
  • Um ein eigenes Burgenpaket mit Hilfe des Patchers zu installieren, kannst du einen Unterordner mit dem Namen deines Pakets im Ordner resources\aiv anlegen. Beschreibungen fügst du in diesem Unterordner als <Sprache>.txt-Dateien ein, wobei <Sprache> für die Sprache steht, in der deine Beschreibung geladen wird (z.B. English.txt).
  • Immer wenn Burgenpakete installiert werden, wird das bereits vorhandene Burgenpaket als Unterordner mit der aktuellen Zeit als Ordnername abgespeichert. Diese Unterordner verbleiben im aiv-Ordner. Allerdings kann bei der Deinstallation des UCP aufgrund momentaner Einschränkungen nur das zuletzt gesicherte Burgenpaket wiederhergestellt werden. Wenn du mehrmals hintereinander AIVs installiert hast, kannst du die jeweiligen Dateien aus den Unterordnern selbst wiederherstellen.

AIC-Dateien

  • Auswahl

    • In der Version 2.12 des UCP konntest du mehrere AIC-Dateien gleichzeitig zur Installation auswählen. Dies wurde in Version 2.13 entfernt. Die Version 2.14 bringt nun diese beliebte Funktion mit einer Verbesserung zurück: Falls du mehrere AICs auswählst, in denen die selben KI-Charaktere enthalten sind, werden diese Charaktere in der Reihenfolge priorisiert, in der du die AICs ausgewählt hast. Ganz konkret bedeutet dies, dass bei einer Mehrfachauswahl die Charaktere der zuerst gewählten AIC in die Installation übernommen werden, während Charaktere, die ignoriert werden, in einer Warnung neben den betroffenen AICs angezeigt werden.

  • Format

    • Das Format der AIC-Dateien wurde geändert. Statt wie bisher in einem benutzerdefinierten Format mit der Dateiendung.aic werden KI-Charaktere nun mit dem JSON-Format mit der Dateiendung .json definiert. Dieses Format ist flexibler, bekannter und einfacher zu warten. Es bereitet den Weg für Erweiterungen in zukünftigen Versionen des UCP.
    • Du kannst AIC-Dateien in den Patcher laden, indem du sie in den Ordner resources\aic einfügst.
    • Du kannst deine alten, in der Version 2.13 unterstützten .aic-Dateien ebenfalls in den Ordner resources\aic einfügen. Diese werden im Patcher zunächst durchgestrichen und mit einer roten Warnung angezeigt. Durch Anklicken der Warnung wird dir eine Konvertierung der Datei in das neue Format angeboten, welches sich anschließend im Patcher verwenden lässt. Die Dateiendung der originalen Datei wird dabei in .aic.bak geändert.
    • Solltest du viele AIC-Dateien besitzen, so kannst du alternativ auch den AICUpdater herunterladen und alle AIC-Dateien, die du konvertieren willst, mit diesem in einen Ordner packen. Durch Doppelklick führst du den AICUpdater aus, welcher dann diese Dateien im selben Ordner für dich konvertiert.
    • Falls beim Konvertieren oder Laden deiner AIC-Dateien Fehler auftreten sollten, ist noch nicht alles verloren. Ein englisches Fehlerprotokoll, welches erklärt, was genau schief gelaufen ist, wird im Ordner mit dem Patcher angelegt. Falls das zum Finden der Fehlerursachen nicht ausreicht, frage doch einfach mal im #help-and-issues-Kanal auf dem offiziellen UCP Discord Server nach.

Änderungen am Patcher:

  • Änderungen an Aufbau und Inhalten

    • Diese Version des Inoffiziellen Crusader Patch führt eine neue Ordnerstruktur ein. Alle Content-Dateien (inklusive AIC- und AIV-Dateien), die vom Patcher verwendet/installiert werden, befinden sich jetzt in einem resources\<config type>-Unterordner im Patcher-Verzeichnis. Beispielsweise kommen AIC-Dateien in den resources\aic-Ordner.
    • Alle Content-Dateien, welche Einstellungen für Spielinhalte enthalten und als Zusammenfassung dieser Einstellungen im Patcher geladen werden können (z.B. AIC-Dateien), sind nun auf das JSON-Format umgestellt.
    • AIC- and AIV-Pakete, die mit dem UCP bereitgestellt werden, wurden auf die jeweils neueste Version aktualisiert.

  • Sprachenunterstützung

    • Es wurde ein Problem beim Austausch von Patchereinstellungen mit Hilfe der ucp.cfg-Datei behoben, welches auftrat, falls die gleiche Datei auf Geräten mit unterschiedlichen Systemsprachen genutzt wurde. Beachte bitte, dass alle Dezimalzahlen mit Nachkommastellen nun durch einen Punkt (.) getrennt werden, um korrekt gelesen werden zu können.
    • Die Sprachen Chinesisch und Ungarisch wurden dem Sprachenkatalog des Patchers hinzugefügt.

  • Es gibt nun die Möglichkeit, den Patcher per Kommandozeile in bereits bestehende Mods einzubinden oder einfach mit bestehenden Einstellungen zu installieren. Weitere Informationen zur Benutzung des Kommandozeilen-Tools findest du in den Installationsschritten.

  • Wir haben einige interne Prozesse des Patchers überarbeitet, um die Benutzung noch einfacher und angenehmer zu gestalten.

  • Wir haben sämtliche Ostereier versteckt.

Changelog

The following changes and improvements have been added with version 2.14 of the Unofficial Crusader Patch.

We now have a website and a discord server!

Notes for multiplayer:

  • The options for changing color of player 1 and activating spectator mode cannot be used in multiplayer.
  • All other settings can be used in multiplayer provided all players use the same configuration.

New Features and Improvements:

General

  • Default multiplayer speed

    • This option sets a default speed when you create a multiplayer lobby. This default will also stay after a match ends and when you go to create another game.

  • Armory/marketplace weapon order fix

    • The weapons icons in armory and marketplace have been updated to have the same order. This option also moves leather armor between mace and crossbow, and moves metal armor between polearm and sword for easier access while navigating items in the marketplace.

  • Ladderman fixes

    • Both actual and displayed recruiting cost of laddermen were fixed to match the intended value of 20 gold (used to show 4 gold and cost 38 gold).
    • AI will now properly use laddermen in sieges independent of whether or not their own keep is currently enclosed.

  • Fire ballista fix

    • Fire ballistae were previously fixed so that they could auto-target monks. This option has been expanded so that they can auto-target tunnelors as well now.

  • Player color changes

    • The ability to change the color of player 1 was previously introduced. However there were a couple of UI elements that the patcher was not able to change it for. These have now been updated to also reflect the color change.

  • Identity menu starting troop selection

    • Version 2.14 of the UCP introduces the ability to override your ranged and melee starting unit selection in-game. This can be accessed from the choose identity menu.
    • Please note that this does currently only affect the Normal gamemode, but not Crusader or Deathmatch ones, and is only active until the game is exited.

  • Strongholdify

    • For all old-school Stronghold lovers and those used to the production chains of Stronghold 1, this option changes the game to scale back the inflated production of Stronghold Crusader to Stronghold 1 settings.
    • This applies to the popularity bonuses from church and cathedral and ale coverage, peasant spawn rates, as well as resource production quantities (e.g. a woodcutter delivers 12 wood instead of 18).

Customizable Starting Troops and Lord Strength

Note for multiplayer: This option can be used in multiplayer, but all players must have the exact same configuration or you will get desyncs or crashes!

  • Starting Troops

    • In the new StartTroops tab, you will be able to select a starting unit configuration for all AI Lords as well as your Crusader or Arabic Lord.
    • The vanilla configuration as well as a UCP patch configuration for starting troops are provided for your convenience.
    • To add your own, you may export and modify these, or add your own into a subfolder of the patcher.
    • Exported starting troops files go into the resources\troops\exports folder. Your own start troop configurations (in JSON format and with .json file extension) can be put into resources\troops.

  • Lord Strength

    • Within the starting troops file, you can also modify the strength, type, and also the dot configurations of the AI Lords. The strength is specified as a multiplier against a base health value of 150000 (same as human player Lord). Dots can be either Blue, Yellow, or None and their count can be 0 to 5.

Customizable Starting Goods

Note for multiplayer: This option can be used in multiplayer, but all players must have the exact same configuration or you will get desyncs or crashes!

  • In the new StartGoods tab, you will be able to select a starting resource configuration that will apply for yourself and all other human or AI players.
  • The vanilla configuration is provided for your convenience. You may export and modify it to create a custom selection.
  • Exported starting goods files go into the resources\goods\exports folder. Your own start goods configurations (in JSON format and with .json file extension) can be put into resources\goods.

AIV Installation

  • Always backup your vanilla (or custom) AIV files before installing!
  • You are able to now load AIV sets into the patcher by creating a subfolder with the name of your AIV set inside the resources\aiv folder. Descriptions for the AIV set go into a <language>.txt file where <language> specifies the language (i.e. English.txt).
  • Each time AIVs are installed, the existing set will be backed up to a timestamped folder. These folders will remain in the aiv folder of your Stronghold Crusader installation. However, please note that due to current limitations of the patcher, when uninstalling the UCP, only the most recent backup will be restored. If you have installed AIVs multiple times you will need to restore the files manually.

AIC Files

  • Selection

    • In version 2.12 of the UCP you were able to select multiple AIC files for installation. In version 2.13, this functionality was removed. With version 2.14, we’re now re-introducing that ability, complete with an upgrade: If you select multiple AICs that have the same characters defined, the one you select first will be given priority and the other ones will show a warning indicating which AI Character definitions will not be used.

  • Format

    • The format of AIC files has changed. Instead of the old custom format with .aic file extension, the patcher is now using a JSON format with .json file extension to specify AI Character definitions. This format is more flexible, widespread and easier to use going forward.
    • You can load AIC files for use by the patcher by placing them into the resources\aic subfolder.
    • You can place your (legacy) .aic files into the resources\aic folder. You will then see a warning in the patcher. If you click the warning, it will offer to convert the AIC file to the newer format so you can install it; the original file will be renamed to a .aic.bak file extension.
    • If you have many AIC files, you can alternatively download the AICUpdater tool and place all AIC files to be converted into a folder with it, and then double-click the AICUpdater to run it.
    • If you encounter errors when trying to convert or load AICs, all is not lost! An error log explaining what went wrong will be created beside the patcher. If that’s not enough help, feel free to ask in the #help-and-issues channel on the official UCP discord server.

Changes to the Patcher:

  • Changes regarding structure and content

    • In this version, the Unofficial Crusader Patch comes with a new folder structure. All content files (including AIC and AIV files) that are to be used/installed by the patcher go into a resources\<config type> subfolder inside the patcher directory. For example your AIC files will go into the resources\aic folder.
    • Any Patcher configuration files like the AIC files are now in JSON format and with .json file extension. Old AIC files can be converted by the patcher.
    • AIC and AIV sets bundled with the UCP have been updated to the newest versions.

  • Language support

    • An issue where sharing ucp.cfg (config) files between persons with different locales would cause crashes or otherwise not work properly has been resolved. Please note that any decimal values in the ucp.cfg must now use a dot (.) as the decimal separator to be interpreted correctly.
    • Translations for Chinese and Hungarian have been added.

  • There is now a command line tool that can be used for quicker or custom installation (see installation steps for details).

  • We’ve made changes and improvements to how some parts of the patcher work internally to make the process more convenient and reliable for you.

  • We’ve hidden all the easter eggs.
